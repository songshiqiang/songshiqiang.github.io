DBA线上安全生产原则

Posted by songshiqiang on 2018-06-16

任何更新操作之前先检查备份是否正常

  • 通过inception之类的上线审核工具是提前生成回滚语句的,但是好像不是每次都成功生成回滚语句
  • 有些mysql分支mysqlbinlog支持–flashback参数,即支持回滚操作
  • mysql binlog_format是row,binlog_row_image = full的时候,解析出binlog,通过sed替换或者binlog2sql等工具也能回滚

但是不是每个人都符合上面的情况,通过备份恢复是最安全的

删除冗余数据库或者表的时候,主从操作要有一定的时间间隔

有时,数据库拆分出去了,我们要删除这些不在使用的冗余数据库

先在一个从库上安全删除数据库,并且不记录binlog

然后过半天,观察主从同步没有问题,在删除其余从库和主库

线上重要操作一定要Double Check

比如上面的删库操作,一定要有人double check,防止本来要删除DB1,但是手误删除了DB2
