Resource-temporarily-unavailable

Posted by songshiqiang on 2018-04-14

This document is not completed and will be updated anytime.

一次线上切proxy,proxy切换到新集群后发现机器卡住了,控制台出现

  • 问题分析
    实际上出现这种现象，本质原因是Linux操作系统无法创建更多进程，导致出错。
    知道了问题原因，那么就让Linux允许创建更多的进程就可以了。

  • 修改Linux最大进程数
    在服务器上执行ulimit -a

    其中open files (-n) 1024 这个值，就是我们要修改的地方了。

  • 问题解决

    我们可以使用 ulimit -u 4096 修改max user processes的值，但是只能在当前终端的这个session里面生效，重新登录后仍然是使用系统默认值。

    正确的修改方式是修改/etc/security/limits.d/90-nproc.conf文件中的值。

具体可以参考这篇文章

https://access.redhat.com/solutions/22105

其实问题发生的时候,我们比对了一下正常机器和故障机器之间配置等有没有不同的地方

我感觉这是一个好思路

可以使用pt-sumary生成系统统计报告,比对一下2台机器系统报告之间有什么不同

另外我们在切换主库的时候,一般通过pt-config-diff比对一下老主库和新主库之间配置有什么不同

比较常见的可能max_connection不同
