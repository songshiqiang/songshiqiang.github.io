Posted by songshiqiang on 2018-11-15

备份导致删库后磁盘空间不释放

收到报警,磁盘空间小于%2

一顿查找,发现有些已经拆分出去不在使用的数据库占用了很大的磁盘空间

那就开始删除这些数据库吧

drop database drop table …….

删除完了,在看一看磁盘空间大小

咦,怎么还是98%,刚刚删除的明明有500G数据嘛

怎么回事?

哈哈,看到这里,经验丰富的应该知道了,这些数据库被别的进程占用了,最有可能的进程就是备份了

停掉备份进程后,磁盘空间立马下降

衍生几个其他的问题

删除general_log磁盘空间不释放

还是磁盘空间不足,发现是general_log占用了很大的空间

直接这么干,磁盘空间还是不会减少
